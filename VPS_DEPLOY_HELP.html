0. Что будет запущено

  • Инфраструктура через Docker Compose: PostgreSQL, Redis, MinIO.
  • API (NestJS) через PM2 на порту 3000.
  • Web (Next.js) через PM2 на порту 3001.
  • Nginx как reverse proxy: /api -> API, остальное -> WEB.

5. Продовый docker-compose для инфраструктуры

Создать файл /opt/escort-platform/docker-compose.prod.yml:

services:
  postgres:
    image: postgres:16-alpine
    container_name: escort-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: 5nR%tVXEdHg0
      POSTGRES_DB: companion_db
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "127.0.0.1:5432:5432"

  redis:
    image: redis:7-alpine
    container_name: escort-redis
    restart: unless-stopped
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    ports:
      - "127.0.0.1:6379:6379"

  minio:
    image: minio/minio:latest
    container_name: escort-minio
    restart: unless-stopped
    environment:
      MINIO_ROOT_USER: companion_minio_admin
      MINIO_ROOT_PASSWORD: 5nR%tVXEdHg0prod
    command: server /data --console-address ":9001"
    volumes:
      - minio_data:/data
    ports:
      - "127.0.0.1:9000:9000"
      - "127.0.0.1:9001:9001"

  minio-init:
    image: minio/minio:latest
    container_name: escort-minio-init
    depends_on:
      - minio
    entrypoint: >
      /bin/sh -c "
      sleep 5;
      mc alias set local http://minio:9000 companion_minio_admin 5nR%tVXEdHg0prod;
      mc mb local/escort-media --ignore-existing;
      mc anonymous set download local/escort-media;
      echo 'MinIO bucket ready';
      "
    restart: "no"

volumes:
  postgres_data:
  redis_data:
  minio_data:

Запуск:

cd /opt/escort-platform
docker compose -f docker-compose.prod.yml up -d

9. Nginx reverse proxy

Создать /etc/nginx/sites-available/escort:

server {
    listen 80;
    server_name 45.9.40.37;

    client_max_body_size 50m;

    location /api/ {
        proxy_pass http://127.0.0.1:3000/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Активация:

ln -sf /etc/nginx/sites-available/escort /etc/nginx/sites-enabled/escort
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl restart nginx
systemctl enable nginx

13. Troubleshooting

API не стартует

pm2 logs escort-api --lines 200
cat /opt/escort-platform/.env

Проверить обязательные env из validation.schema.ts (JWT, ENCRYPTION_KEY, DATABASE_URL и т.д.).

Web не стартует

pm2 logs escort-web --lines 200
cd /opt/escort-platform/apps/web && npm run start

Nginx 502

ss -tulpn | grep -E ":3000|:3001"
nginx -t
journalctl -u nginx -n 100 --no-pager

DB подключение не работает

docker logs escort-postgres --tail 100
docker exec -it escort-postgres psql -U postgres -d companion_db -c "select now();"

14. Security hardening minimum

  • Сменить root password после первого логина.
  • Создать отдельного deploy-пользователя, отключить root ssh login и password auth (после настройки ключей).
  • Никогда не коммитить реальные пароли/секреты в git.
  • Ограничить внешние порты: оставить только 22/80/443.

Если секрет уже попал в переписку/репозиторий - считать его скомпрометированным и заменить.
