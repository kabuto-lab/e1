A1 Маппинг «City» → hairColor BUG ›

Проблема: В формах создания и редактирования модели поле с лейблом «Город» записывает значение в physicalAttributes.hairColor. Это приводит к порче данных — цвет волос перезаписывается названием города.

Причина: Неправильный register() path в react-hook-form. Строка ~360 в edit/page.tsx привязывает поле City к physicalAttributes.hairColor.

Решение:

// Было (НЕПРАВИЛЬНО): {...register('physicalAttributes.hairColor')} label="Город" // Нужно одно из двух: // 1. Добавить поле city в physical_attributes (схема БД) {...register('physicalAttributes.city')} label="Город" // 2. Или убрать поле City и добавить отдельное поле для hairColor {...register('physicalAttributes.hairColor')} label="Цвет волос"
  • apps/web/app/dashboard/models/[id]/edit/page.tsx (строка ~360)
  • apps/web/app/dashboard/models/create/page.tsx (аналогичный баг)
  • packages/db/src/schema/model-profiles.ts (если добавлять city в схему)
Рекомендация: добавить city как отдельное поле в physical_attributes JSON, а hairColor вернуть на своё место с правильным лейблом.
A2 Валидация отключена на странице Edit BUG ›

Проблема: zodResolver импортирован, но не подключён к useForm(). Форма принимает любые данные без проверки — можно отправить пустое имя, отрицательный возраст, невалидный slug.

Причина: В вызове useForm() на строке ~63 указан только mode: 'onSubmit', но нет resolver: zodResolver(schema).

Решение:

const editProfileSchema = z.object({ displayName: z.string().min(2, 'Минимум 2 символа').max(100), slug: z.string().min(2).max(100).regex(/^[a-z0-9-]+$/), biography: z.string().max(2000).optional(), physicalAttributes: z.object({ age: z.coerce.number().min(18).max(70), height: z.coerce.number().min(140).max(200), weight: z.coerce.number().min(40).max(120), // ... остальные поля }), rateHourly: z.coerce.number().min(0).optional(), rateOvernight: z.coerce.number().min(0).optional(), }); const form = useForm({ mode: 'onSubmit', resolver: zodResolver(editProfileSchema), });

G4 Token refresh в AuthProvider SECURITY ›

Проблема: AuthProvider хранит JWT access token в localStorage, но не проверяет его срок действия и не обновляет. Access token живёт 15 минут (по схеме JWT в бэкенде). После 15 минут все запросы начнут получать 401.
