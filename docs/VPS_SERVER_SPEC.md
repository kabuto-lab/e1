# VPS — снимок спецификации (операторский)

**Срез:** по выводу команд на сервере (**апрель 2026**, конфигурация после апгрейда ВМ).  
**Назначение:** ориентир для деплоя, оценки ресурсов и рисков (память, диск, Docker).

**Публичный IPv4** в этом файле намеренно не дублируется — храните у себя вне репозитория или в закрытой заметке.

## Риски и выводы для этого стека (Nest + Next + Postgres в Docker)

1. **Память:** конфигурация **значительно комфортнее** для `npm ci` / `npm run build`, API + Next + контейнеров. **Swap = 0** — при редком пике памяти возможен **OOM** вместо замедления через swap; для прод-ВМ это часто приемлемо, но имейте в виду при тяжёлых параллельных сборках.
2. **Диск 40 GB** — запас для образов Docker и артефактов лучше; всё равно периодически `docker system prune` и контроль логов.
3. **PM2 `escort-api`** (`max_memory_restart: 512M` в `ecosystem.config.cjs`) — по-прежнему разумный лимит; общий запас RAM теперь позволяет стабильнее держать Next и Postgres.
4. **Node ≥ 22** по `package.json` — убедитесь, что на сервере после апгрейда образа/ручной установки по-прежнему **Node 22+** (`node -v`).
