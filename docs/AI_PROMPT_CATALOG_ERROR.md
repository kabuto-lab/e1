### Root Cause of the Recurring Error (this is what actually happened last time)

Even when `check:postgres-env` says "OK", the password stored inside the Postgres Docker volume can still be different from the one in .env. Only `ensure:database` performs a real connection and automatically runs `ALTER ROLE postgres WITH PASSWORD ...` when it detects 28P01 and the container is running.
