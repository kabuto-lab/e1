1. CANVAS RESOLUTION CAP (CRITICAL - #1 FPS KILLER)
--------------------------------------------------------------------------------

BEFORE:
  canvas.width = window.innerWidth * window.devicePixelRatio
  // On 4K monitor with DPR=2: 7680 × 4320 = 33 MILLION PIXELS

AFTER:
  const MAX_DPR = 1.5;  // Hard cap
  const MAX_WIDTH = 1920;  // 1080p max
  const MAX_HEIGHT = 1080;
  const MAX_PIXELS = 2_073_600;  // 2MP limit
  
  const dpr = Math.min(window.devicePixelRatio || 1, MAX_DPR);
  let canvasWidth = Math.floor(window.innerWidth * dpr);
  let canvasHeight = Math.floor(window.innerHeight * dpr);
  
  // Emergency downscale if too big
  if (canvasWidth * canvasHeight > MAX_PIXELS) {
    const scale = Math.sqrt(MAX_PIXELS / (canvasWidth * canvasHeight));
    canvasWidth = Math.floor(canvasWidth * scale);
    canvasHeight = Math.floor(canvasHeight * scale);
  }

RESULT: 33M pixels → 2M pixels (16× LESS WORK)

================================================================================
WHY IT WAS LAGGING (ROOT CAUSE ANALYSIS)
================================================================================

THE KILLER: Canvas at Full Native Resolution

On a 4K monitor (3840×2160) with DPR=2 (MacBook Retina):
  canvas.width = 3840 × 2 = 7680
  canvas.height = 2160 × 2 = 4320
  Total pixels = 7680 × 4320 = 33,177,600 (33 MILLION)

The fragment shader runs ONCE PER PIXEL.

33 MILLION shader executions PER FRAME.

At 60 FPS target:
  33M × 60 = 1.98 BILLION shader executions per second

No consumer GPU can handle this.

THE FIX:
  Cap DPR to 1.5: 3840 × 1.5 = 5760, 2160 × 1.5 = 3240
  Emergency downscale: sqrt(2M / 18M) = 0.33
  Final: 5760 × 0.33 = 1920, 3240 × 0.33 = 1080
  Total: 1920 × 1080 = 2,073,600 (2 MILLION)

2M × 20 FPS = 40 MILLION shader executions per second

1.98B → 40M = 50× LESS GPU WORK
