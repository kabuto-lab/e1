🔴 PHASE 1: Critical Security (BLOCKING)

Week 1 — Blocking

Критические исправления безопасности. Блокируют production deployment.

  • 1.1 JWT Authentication Guard
    Реализовать полноценный JWT guard с валидацией токена, проверкой blacklist в Redis, продлением сессии и обработкой ошибок (expired, invalid, revoked).
    📁 apps/api/src/auth/guards/jwt-auth.guard.ts
  • 1.2 RBAC Roles Guard
    Guard с проверкой ролей (ADMIN > MANAGER > MODEL > CLIENT), иерархия доступов, проверка ownership ресурсов.
    📁 apps/api/src/auth/guards/roles.guard.ts
  • 1.3 Environment Validation Schema
    Zod схема для валидации .env при старте приложения. Блокировка запуска при отсутствии JWT_SECRET, ENCRYPTION_KEY.
    📁 apps/api/src/config/validation.schema.ts
  • 1.4 Rate Limiting & DDoS Protection
    @nestjs/throttler с разными лимитами для auth (5/min), API (100/min), строгих операций (10/min).
    📁 apps/api/src/security/rate-limit.config.ts

🟠 PHASE 2: Input Validation

Week 1-2
  • 2.1 Enhanced DTO Validation
    class-validator + Zod схемы для всех DTO. Валидация email, паролей (8+ символов, спецсимволы), физических параметров (age 18-45, height 150-200).
    📁 apps/api/src/profiles/dto/*.dto.ts
  • 2.2 SQL Injection Prevention
    Whitelist sortable columns, валидация limit/offset, параметризованные запросы Drizzle ORM.
    📁 apps/api/src/profiles/profiles.service.ts

🟡 PHASE 3: Observability

Week 2
  • 3.1 Audit Logging Service (152-ФЗ)
    Логирование всех критических событий: login, profile operations, bookings, payments. Маскирование IP, sanitization данных.
    📁 apps/api/src/audit/audit-logger.service.ts

🟢 PHASE 4: Performance

Week 2-3
  • 4.1 Multi-Layer Caching
    L1 (Map) + L2 (Redis) кэширование. TTL: 60s L1, 300s L2. Tag-based invalidation.
    📁 apps/api/src/cache/cache.service.ts
  • 4.2 Helmet Security Headers
    CSP, HSTS, XSS Filter, Referrer Policy. Настройка для Next.js + Three.js.
    📁 apps/api/src/security/helmet.config.ts
  • 4.3 Anti-Leak Communication System
    Детекция и маскирование телефонов, email, соцсетей в сообщениях. Блокировка при 3 нарушениях.
    📁 apps/api/src/communications/anti-leak.service.ts

⚡ Quick Start Commands

📦 Установка зависимостей: 
cd apps/api
npm install @nestjs/jwt @nestjs/passport passport passport-jwt
npm install @nestjs/throttler ioredis cache-manager
npm install zod nestjs-zod
npm install helmet dompurify jsdom
npm install class-validator class-transformer
🔐 Генерация секретов: 
# Windows PowerShell
$JWT_SECRET = -join ((48..57 + 65..70 + 97..102) | Get-Random -Count 64 | ForEach-Object {[char]$_})
$ENCRYPTION_KEY = -join ((48..57 + 65..70 + 97..102) | Get-Random -Count 64 | ForEach-Object {[char]$_})

echo "JWT_SECRET=$JWT_SECRET" >> .env
echo "ENCRYPTION_KEY=$ENCRYPTION_KEY" >> .env
🚀 Запуск разработки: 
# Terminal 1 - Database
docker-compose -f docker-compose.dev.yml up -d

# Terminal 2 - API
cd apps/api
npm run start:dev

# Terminal 3 - Web
cd apps/web
npm run dev
