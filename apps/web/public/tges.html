Схема 1 · Manual Escrow

Эскроу без банка

Платформа не хранит и не движет деньгами — только ведёт учётный контур в БД. Истина по оплате — ручная проверка менеджером (скрин / выписка / договорённость). Telegram — канал уведомлений. Идеально как MVP перед подключением банка или крипто-эскроу.

Юридические риски
⚠
Приём фиата на счёт юрлица без лицензии ЦБ РФ может квалифицироваться как платёжная деятельность (ст. 3, 161-ФЗ). Перед запуском — обязательно с юристом. Не писать на сайте: «мы гарантируем», «холдим деньги», «эскроу-счёт». Писать: «Платформа ведёт учётный контур. Деньги передаются напрямую между сторонами».
◈
Вариант подходит для пилота / внутреннего запуска. Для массового публичного потока — банк (Т-Банк / ЮKassa, см. escrow.html) или отдельно проработанное крипто-эскроу (вкладка 06).

EscrowService — ядро
TypeScript · NestJS@Injectable()
export class EscrowService {
  async createOrGet(bookingId: number, amount: number) {
    const existing = await this.db.query.escrowTransactions.findFirst({
      where: eq(escrowTransactions.bookingId, bookingId),
    });
    if (existing) return existing;
    return this.db.insert(escrowTransactions).values({
      bookingId,
      amount: amount.toString(),
      paymentProvider: 'manual',
      status: 'pending_funding',
    }).returning();
  }

  async fund(id: number, managerId: number, metadata?: Record<string, unknown>) {
    return this.db.transaction(async (tx) => {
      const escrow = await tx.query.escrowTransactions.findFirst({
        where: eq(escrowTransactions.id, id),
      });
      if (!escrow) throw new NotFoundException();
      if (escrow.status !== 'pending_funding')
        throw new BadRequestException('Неверный статус эскроу');
      await tx.update(escrowTransactions)
        .set({ status: 'funded', fundedAt: new Date(), managerId, metadata })
        .where(eq(escrowTransactions.id, id));
      await tx.update(bookings)
        .set({ status: 'escrow_funded' })
        .where(eq(bookings.id, escrow.bookingId));
      this.events.emit('escrow.funded', { escrowId: id, bookingId: escrow.bookingId });
    });
  }

  async release(id: number) { /* funded → released, booking → completed */ }
  async refund(id: number)  { /* … */ }
}
Защита эндпоинтов
TypeScript@Controller('escrow')
@UseGuards(JwtAuthGuard)
export class EscrowController {
  @Post()
  create(@Body() dto: CreateEscrowDto) {
    return this.escrowService.createOrGet(dto.bookingId, dto.amount);
  }

  @Post(':id/fund')
  @Roles('manager', 'admin')
  @UseGuards(RolesGuard)
  fund(@Param('id') id: number, @CurrentUser() manager: User, @Body() body: { metadata?: any }) {
    return this.escrowService.fund(id, manager.id, body.metadata);
  }

  @Get('by-booking/:bookingId')
  getByBooking(@Param('bookingId') bookingId: number, @CurrentUser() user: User) {
    return this.escrowService.getByBookingId(bookingId, user.id);
  }
}

Схема 3 · ориентир

Крипто без банка

Отдельная ветка продукта: ончейн-адрес, индексатор, не путать с учётным manual.

⚠
Не юридическая консультация. Крипто и фиат в РФ — отдельные риски (ЦФА, 115-ФЗ, санкции). Перед запуском — юрист и compliance.

Разблокировка по смарт-контракту или мультисиг 2-of-3; бот шлёт адрес/QR; funded после подтверждения индексатором. Провайдер вроде tron_usdt или отдельная таблица сделок.
