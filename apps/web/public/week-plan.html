Escrow E2E тесты
Happy path + 4 негативных сценария
  • 🧪
    E2E happy path — release
    createIntent → recordDeposit → funded → confirmRelease → released
    E2E
  • 🧪
    E2E happy path — refund
    createIntent → funded → confirmRefund → refunded
    E2E
  • ⛔
    Негатив: double-spend (replay tx_hash)
    Повторный tx_hash → 409 idempotent, не создаёт второй депозит.
    Негатив
  • ⛔
    Негатив: wrong memo / amount mismatch
    Неверный memo не привязывается. Частичная сумма → partially_funded.
    Негатив
  • ⛔
    Негатив: release/refund из неверного статуса
    Попытка release при pending_funding → 422 Unprocessable Entity.
    Негатив
