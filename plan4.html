🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ ПРЕДЫДУЩЕГО ПЛАНА

  1. Fake Token — НЕ РАБОТАЕТ
    Подпись dGVzdC1zaWduYXR1cmUtZm9yLWRlbW8tcHVycG9zZXM — это base64 от строки "test-signature-for-demo-purposes", а не валидная JWT подпись. NestJS отклонит этот токен при проверке секрета.
  2. Отключение UsersModule — НЕ РЕШЕНИЕ
    AuthService использует UsersModule для валидации пользователей. Без него login вообще не будет работать.
  3. Пропущена главная причина 404
    План не объясняет почему AuthController не регистрируется. Это не конфликт модулей — скорее всего проблема в компиляции TypeScript или ошибке при старте API.

ОШИБКА #2: 401 Invalid token format

ДЕТАЛЬНЫЙ АНАЛИЗ:

// При попытке создать модель: ❌ API Error Response Body: { "statusCode": 401, "message": "Invalid token format", "stack": "UnauthorizedException: Invalid token format at JwtAuthGuard.canActivate (jwt-auth.guard.ts:86:15)" }

ВОЗМОЖНЫЕ ПРИЧИНЫ (по приоритету):

  1. ВЫСОКАЯ Токен пустой (null) — пользователь не залогинен
  2. ВЫСОКАЯ Токен с кавычками — "eyJhbG..." вместо eyJhbG...
  3. СРЕДНЯЯ Токен содержит "Bearer" — двойной префикс
  4. СРЕДНЯЯ Токен повреждён — не 3 части JWT

3Исправление api-client.ts (10 минут)

ЦЕЛЬ:

Очистить токен от типичных проблем (кавычки, Bearer префикс, валидация формата).

ДЕЙСТВИЯ:

  1. Открыть файл: apps/web/lib/api-client.ts
  2. Найти функцию getAuthHeader()
  3. Заменить на:
    function getAuthHeader(): HeadersInit { let token = localStorage.getItem('accessToken'); if (!token) { console.log('🔑 No token in localStorage'); return {}; } // Очистка от типичных проблем token = token.trim(); token = token.replace(/^["']|["']$/g, ''); // Удалить кавычки token = token.replace(/^Bearer\s+/i, ''); // Удалить "Bearer " // Валидация формата JWT (должно быть 3 части) const parts = token.split('.'); if (parts.length !== 3) { console.warn('❌ Invalid JWT format:', parts.length, 'parts instead of 3'); console.warn('Token:', token.substring(0, 50) + '...'); return {}; } console.log('🔑 Token OK:', token.substring(0, 20) + '...'); return { Authorization: `Bearer ${token}` }; }
  4. Сохранить файл
  5. Обновить страницу (Ctrl+Shift+R)

ЧТО ЭТО ИСПРАВЛЯЕТ:

  • ✅ Токен с кавычками: "eyJhbG..." → eyJhbG...
  • ✅ Токен с Bearer: "Bearer eyJhbG..." → eyJhbG...
  • ✅ Повреждённый токен: возвращает пустой заголовок вместо ошибки
  • ✅ Логирование для отладки

4Исправление JwtAuthGuard (15 минут)

ЦЕЛЬ:

Упростить и сделать надёжной валидацию токена.

ДЕЙСТВИЯ:

  1. Открыть файл: apps/api/src/auth/guards/jwt-auth.guard.ts
  2. Найти метод canActivate()
  3. Заменить на:
    @Injectable() export class JwtAuthGuard implements CanActivate { constructor( private jwtService: JwtService, private configService: ConfigService, ) {} async canActivate(context: ExecutionContext): Promise<boolean> { const request = context.switchToHttp().getRequest(); const token = this.extractTokenFromHeader(request); if (!token) { console.log('❌ No token in Authorization header'); throw new UnauthorizedException('Authorization header missing or invalid'); } try { const payload = await this.jwtService.verifyAsync(token, { secret: this.configService.get('JWT_SECRET'), }); request['user'] = payload; console.log('✅ JWT verified for user:', payload.userId); return true; } catch (error) { console.error('❌ JWT verification failed:', error.message); throw new UnauthorizedException('Invalid or expired token'); } } private extractTokenFromHeader(request: Request): string | undefined { const authHeader = request.headers.authorization; if (!authHeader) return undefined; const [type, token] = authHeader.split(' '); return type === 'Bearer' ? token : undefined; } }
  4. Сохранить файл
  5. Перезапустить API сервер:
    # В терминале API: Ctrl+C npm run start:dev

ЧТО ЭТО ИСПРАВЛЯЕТ:

  • ✅ Упрощённая логика — меньше мест для ошибок
  • ✅ Ясное логирование — видно что происходит
  • ✅ Правильная обработка ошибок — понятные сообщения

5Проверка AuthModule (если 404 сохраняется)

ЦЕЛЬ:

Убедиться что AuthModule правильно настроен и импортирован.

ДЕЙСТВИЯ:

  1. Открыть файл: apps/api/src/auth/auth.module.ts
  2. Проверить что выглядит так:
    @Module({ imports: [ UsersModule, // ← Обязательно для AuthService JwtModule.registerAsync({ imports: [ConfigModule], useFactory: (configService: ConfigService) => ({ secret: configService.get('JWT_SECRET'), signOptions: { expiresIn: '1d' }, }), inject: [ConfigService], }), ], controllers: [AuthController], // ← Должен быть здесь providers: [AuthService], exports: [AuthService], }) export class AuthModule {}
  3. Открыть файл: apps/api/src/app.module.ts
  4. Проверить что AuthModule импортирован:
    @Module({ imports: [ ConfigModule.forRoot({ isGlobal: true }), DatabaseModule, AuthModule, // ← ДОЛЖЕН БЫТЬ ModelsModule, // ← Работает, значит система ок // ...остальные модули ], })
  5. Перезапустить API: Ctrl+C → npm run start:dev
  6. Проверить консоль: нет ли ошибок при старте

6Альтернативное решение — Public Endpoint (если всё сложно)

ЦЕЛЬ:

Временно отключить JWT guard для разработки.

ВАРИАНТ A: Отключить guard для создания моделей

# Файл: apps/api/src/models/models.controller.ts @Post() // @UseGuards(JwtAuthGuard) // ← ВРЕМЕННО ЗАКОММЕНТИРОВАТЬ async create(@Body() createModelDto: CreateModelDto) { return this.modelsService.create(createModelDto); }

ВАРИАНТ B: Создать декоратор @Public()

# Файл: apps/api/src/auth/decorators/public.decorator.ts import { SetMetadata } from '@nestjs/common'; export const IS_PUBLIC_KEY = 'isPublic'; export const Public = () => SetMetadata(IS_PUBLIC_KEY, true);
# Файл: apps/api/src/auth/auth.controller.ts @Post('login') @Public() // ← Отключить auth для этого endpoint @HttpCode(HttpStatus.OK) async login(@Body() body: LoginDto) { return await this.authService.login(body.email, body.password); }

⚠️ ВАЖНО:

Это только для локальной разработки! В продакшене верни guard.
