📊 1. Detailed Analysis по Компонентам

1.1 Pre-Moderation Workflow

✅ ВЕРДИКТ: КРИТИЧЕСКИ НУЖНО
Компонент Статус Нужно? Комментарий
moderationStatus поле Частично ✓ Добавить в schema
moderatedBy + moderatedAt Нет ✓ Для аудита
Endpoint PATCH /moderate Нет ✓ Только Admin/Manager
PublishedOnlyGuard Частично ✓ Фильтр approved только
Email уведомления админу Нет ~ Можно отложить на Week 4

💡 Рекомендация:

Pre-moderation — это must-have для премиум платформы. Без этого невозможно запуститься. Email уведомления можно заменить на internal notifications в админке (экономия 2-3 часа).

1.2 Photo Privacy (Presigned URLs)

✅ ВЕРДИКТ: НУЖНО
Компонент Статус Нужно? Комментарий
Private MinIO bucket Нет ✓ Безопаснее public
Presigned URLs (1 час TTL) Нет ✓ Оптимальный выбор
minioKey в БД (не URL) Частично ✓ Правильная архитектура
Auto-refresh при 403 Нет ~ Можно упростить

💡 Рекомендация:

Presigned URLs — правильный выбор. Proxy через API был бы медленнее и создал бы лишнюю нагрузку. Для MVP достаточно TTL 1 час без auto-refresh (пользователь обновит страницу).

1.3 Bookings — Упрощенный Flow

✅ ВЕРДИКТ: ПРАВИЛЬНОЕ РЕШЕНИЕ
Компонент Статус Нужно? Комментарий
Статусы PENDING/CONFIRMED/DECLINED ✓ Есть ✓ Уже в state machine
Без оплаты (эскроу исключен) ✓ Решение ✓ Правильно для MVP
Без чата ✓ Решение ✓ Экономия 20+ часов

💡 Рекомендация:

Исключение эскроу и чата из MVP — отличное решение. Это сокращает scope на ~40 часов. Текущий booking module уже имеет state machine — нужно только упростить UI.

1.4 Debug Panel v2.0

⚠️ ВЕРДИКТ: ОПЦИОНАЛЬНО (Nice-to-have)
Вкладка Сложность Нужно? Комментарий
System Tab (health checks) Средняя ✓ Полезно для отладки
Auth Tab (JWT проверка) Средняя ✓ Критично для dev
Network Tab (лог запросов) Низкая ~ Уже есть в DebugPanel.tsx
Logs Tab (console capture) Высокая ✗ Можно вырезать
Env Tab (переменные) Низкая ~ Полезно, но не критично
FAB + Draggable Modal Средняя ~ UI polish, можно упростить

💡 Рекомендация:

Debug Panel v2 — это nice-to-have, а не must-have. Для MVP достаточно:

  • ✓ System Tab (health checks) — уже работает
  • ✓ Network Tab (request log) — уже работает
  • ✗ Logs Tab — вырезать (экономия 4-6 часов)
  • ✗ Draggable window — оставить fixed position (экономия 3-4 часа)

🛡️ 3. Security Checklist — Анализ

  • Нет endpoint'ов без @UseGuards(JwtAuthGuard)
    ⚠️ Частично: Guards есть, но JwtAuthGuard — заглушка. Нужно исправить (Week 1, Day 1-2)
  • RolesGuard проверяет роль для каждого endpoint'а
    ⚠️ Частично: RolesGuard — пустая заглушка. Нужно реализовать (Week 1, Day 3)
  • Rate limiting работает
    ❌ Не реализован. Нужно настроить @nestjs/throttler (Week 1, Day 4)
  • SQL Injection невозможен (whitelist на сортировку)
    ⚠️ Частично: profiles.service.ts имеет риск. Нужно исправить (Week 1, Day 6)
  • Фото недоступны по прямой ссылке (presigned)
    ❌ Не реализован. Нужно MinIO presigned URLs (Week 2, Day 12)
  • Debug Panel не доступен в production
    ⚠️ Частично: Нет проверки NODE_ENV. Нужно добавить (Week 3, Day 21)
  • Все секреты в .env (не в коде)
    ✓ Реализовано: JWT_SECRET, DATABASE_URL и другие в .env
