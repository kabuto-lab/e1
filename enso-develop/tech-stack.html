🎯 Key Architectural Principles

Polyglot Architecture: Enso combines Scala (backend/runtime), Rust (parser/FFI), TypeScript/Vue.js (frontend), and Java (JVM ecosystem) to leverage the strengths of each language.

JPMS (Java Platform Module System): Full modularization for better dependency management and runtime performance. All Java/Scala components are properly modularized.

CRDT-based Collaboration: Real-time collaborative editing using Yjs CRDT library with WebSocket synchronization.

GraalVM Native Image: Optional native compilation for improved startup time and reduced memory footprint.

💡 Key Technical Decisions & Architecture Patterns

🎯 Polyglot Architecture

Enso uses a carefully chosen polyglot architecture where each language is used for what it does best:

  • Scala: Type-safe, functional backend with excellent JVM ecosystem
  • Rust: High-performance, memory-safe parser and FFI layer
  • TypeScript/Vue.js: Modern, reactive frontend with excellent developer experience
  • Java: GraalVM integration and enterprise ecosystem
  • Enso: Domain-specific dataflow language for end users

🔧 JPMS (Java Platform Module System)

Full modularization using JPMS provides:

  • Better encapsulation and dependency management
  • Improved startup time with custom runtime images
  • Clear module boundaries and APIs
  • Reduced memory footprint

⚡ GraalVM Native Image

Optional native compilation provides:

  • Faster startup times (no JVM warmup)
  • Lower memory footprint
  • Standalone executables
  • Better security (no dynamic class loading)

🔄 CRDT-based Collaboration

Real-time collaborative editing using:

  • Yjs: High-performance CRDT library
  • y-websocket: WebSocket-based synchronization
  • Conflict-free: Automatic merge of concurrent edits
  • Offline-first: Works without network, syncs later
